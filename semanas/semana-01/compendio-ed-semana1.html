Notación Big-O

Definición Formal

"La notación Big-O es una representación matemática que describe cómo crece la complejidad de un algoritmo en función del tamaño de la entrada, generalmente denotado como 'n'."

— Análisis de Algoritmos

Propósito Principal

La notación Big-O es fundamental para programadores porque:

  • Predice el rendimiento de algoritmos en el peor caso
  • Permite comparar eficiencia entre diferentes soluciones
  • Garantiza escalabilidad de sistemas computacionales

Héctor de León (s.f.)

Tipos de Complejidad

O(1) - Constante

Tiempo independiente del tamaño

O(n) - Lineal

Crecimiento proporcional a la entrada

O(n²) - Cuadrática

Crecimiento cuadrático con la entrada

O(log n) - Logarítmica

Crecimiento logarítmico eficiente

Tipos de Datos Abstractos (TDA)

Concepto Fundamental

Campos (2022): "La diferencia estriba en el énfasis que se pretende dar a la especificación formal de los tipos como herramienta fundamental para el diseño modular de programas"

Definición

Modelo matemático definido por un conjunto de datos y operaciones asociadas.

Abstracción de datos y operaciones

Separación Interfaz-Implementación

Interfaz (qué hace): Define operaciones y comportamiento

Implementación (cómo lo hace): Estructura de datos específica

Representación en Memoria

Clasificación Principal

Dos enfoques fundamentales para organizar datos en memoria:

Estática (Arreglos)

Memoria contigua, tamaño fijo

Dinámica (Enlaces)

Punteros, tamaño variable

Impacto en Complejidad

La elección afecta directamente:

Coste Temporal

Arreglo: O(1) acceso
Lista: O(n) búsqueda

Coste Espacial

Arreglo: O(MAX)
Lista: O(n) dinámico

Comparación Práctica

Arreglo contiguo vs Lista enlazada: acceso y organización en memoria

Criterios de Selección

Usar Arreglos cuando:

Acceso frecuente por índice, tamaño conocido

Usar Listas cuando:

Tamaño variable, inserciones/eliminaciones frecuentes

Considerar híbridos:

Árboles balanceados para O(log n) equilibrado

¿Qué es un TDA y cuál es la ventaja de separar interfaz e implementación?

Modelo Matemático

TDA: modelo definido por datos y operaciones. Ventaja: una misma interfaz puede tener múltiples implementaciones, permitiendo flexibilidad y reutilización.

Representación en Memoria

¿Cuáles son las dos formas principales de organizar datos en memoria?

Estática vs Dinámica

Estática (Arreglos): memoria contigua, acceso O(1). Dinámica (Enlaces): punteros, tamaño flexible, acceso O(n).
