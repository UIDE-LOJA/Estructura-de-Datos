1. Introducción a la Notación Big-O

1.1 Definición y Propósito

La complejidad algorítmica es un concepto fundamental en el análisis de algoritmos, diseñado para pronosticar el rendimiento de un algoritmo en relación con el tamaño de los datos de entrada.

Notación Big-O

Es una representación matemática que describe cómo crece la complejidad de un algoritmo en función del tamaño de la entrada, generalmente denotado como "n".

Propósito Principal

Análisis del Peor Caso

Proveer una visión clara de cómo se comportará el algoritmo en el "peor de los casos", es decir, en el escenario que exige la mayor cantidad de operaciones.

Eficiencia y Rendimiento

La notación Big-O mide la eficiencia y el rendimiento de un algoritmo utilizando la complejidad de tiempo y espacio, aunque no indica qué tan rápido es el tiempo de ejecución en términos absolutos.

1.2 Tipos de Complejidad (Asintótica)

Principales tipos de complejidades Big-O:

O(1) - Constante
O(n) - Lineal
Tiempo proporcional al tamaño
O(log n) - Logarítmica
O(n²) - Cuadrática
Crecimiento cuadrático
O(2ⁿ) - Exponencial | O(n!) - Factorial
Complejidades muy costosas

2. Tipos de Datos Abstractos (TDA)

2.1 Definición y Abstracción

El Tipo de Dato Abstracto (TDA) es un modelo matemático que se define en términos de un conjunto de datos u objetos al cual se le asocian una serie de operaciones.

Características Fundamentales:

Abstracción

Se centra en la abstracción de datos

Interfaz

Define qué hace, no cómo lo hace

Encapsulamiento

Oculta detalles de implementación

2.2 Implementación del TDA

La separación entre la interfaz (qué hace) y la implementación (cómo lo hace) es crucial.

Proceso de Implementación:

  1. Encontrar una estructura de datos (ED) adecuada para representar el modelo subyacente
  2. Escribir los procedimientos o funciones que cumplirán con las tareas definidas por las operaciones
Orientación a Objetos

El paradigma de orientación a objetos facilita esto mediante el encapsulamiento de datos y operaciones a través de clases e interfaces.

2.3 Ejemplos de TDA Fundamentales

Muchos de los TADs más importantes son contenedores de datos, es decir, agregaciones de múltiples datos en una unidad conceptual.

Listas

Colecciones secuenciales de elementos

Pilas (Stack)

Colecciones que siguen la propiedad LIFO (Last In, First Out)

Colas (Queue)

Colecciones que siguen la propiedad FIFO (First In, First Out)

Árboles y Diccionarios

Estructuras jerárquicas y conjuntos de pares clave-valor

3. Representación en Memoria

3.1 Clasificación General

Las estructuras de datos se pueden clasificar, según su organización en memoria, como:

No Enlazadas

Arreglos/Vectores

Representación Estática

Enlazadas

Nodos/Punteros

Representación Dinámica

Esta elección tiene implicaciones directas en el coste temporal (Big-O) y espacial de las operaciones.

3.2 Representación Estática (Arreglo/Vector)

El espacio de almacenamiento se conoce en tiempo de compilación y se mantiene reservado durante la ejecución.

Características:

Contigüidad

Los elementos se almacenan en ubicaciones de memoria contiguas

Acceso Directo - O(1)

Coste constante para insertar, modificar, obtener y borrar

Limitación - O(n)

Redimensionamiento requiere copiar todos los elementos

3.3 Representación Dinámica (Nodo/Enlace)

El espacio de almacenamiento se obtiene durante la ejecución, a medida que se necesita.

Características:

Nodos y Punteros

Se usa memoria dinámica con punteros que direccionan células en memoria

Estructura Dinámica

La cantidad máxima de memoria no se conoce hasta finalizar la ejecución

Coste en Tiempo - O(n)

Operaciones como añadir, buscar y borrar en listas enlazadas

3.4 Comparación: Arreglo vs. Enlace

Representación Coste en Espacio Coste en Tiempo (Peor Caso) Flexibilidad
Arreglo/Vector O(MAX) - Fijo O(1) Menos flexible
Lista Enlazada O(n) - Dinámico O(n) Más flexible
Árbol Binario Equilibrado O(n) O(log n) Excelente rendimiento

Ventajas de Cada Representación

Representación Estática

  • Acceso directo O(1)
  • Eficiencia en operaciones básicas
  • Menor overhead de memoria
  • Localidad de referencia

Representación Dinámica

  • Flexibilidad en tamaño
  • No limitación de cardinal máximo
  • Uso eficiente de memoria
  • Facilita inserción/eliminación
