1. Notación Big-O (O Grande)

1.1 Definición y Propósito

La notación Big-O es una representación matemática que describe cómo crece la complejidad de un algoritmo en función del tamaño de la entrada, generalmente denotado como "n".

Propósito Principal

Proveer una visión clara de cómo se comportará el algoritmo en el "peor de los casos", es decir, en el escenario que exige la mayor cantidad de operaciones.

1.2 Tipos de Complejidad Asintótica

Complejidades Principales

O(1) - Constante O(n) - Lineal O(n log n) - Logarítmica
O(n²) - Cuadrática O(2ⁿ) - Exponencial O(n!) - Factorial

1.3 Características de Cada Complejidad

O(1) - Constante

El tiempo de ejecución no cambia con el tamaño de la entrada. Ejemplo: Acceso directo a un elemento de un arreglo.

O(n) - Lineal

El tiempo de ejecución crecerá proporcionalmente al tamaño de la entrada. Ejemplo: Búsqueda secuencial en una lista.

O(n²) - Cuadrática

Un aumento en la entrada resultará en un crecimiento cuadrático en el tiempo. Ejemplo: Algoritmos de ordenamiento básicos.

1.4 Importancia en la Programación

Para Programadores
  • Optimización de códigos
  • Garantizar escalabilidad
  • Mejorar eficiencia
Medición
  • Complejidad temporal
  • Complejidad espacial
  • Rendimiento del algoritmo

2. Tipos de Datos Abstractos (TDA)

2.1 Definición y Concepto

Un Tipo de Dato Abstracto (TDA) es un modelo matemático que se define en términos de un conjunto de datos u objetos al cual se le asocian una serie de operaciones.

Abstracción

El TDA provee una interfaz que permite realizar las operaciones permitidas, abstraiéndose de la manera en que dichas operaciones están implementadas.

2.2 Separación Interfaz-Implementación

Interfaz (Qué hace)
  • Define operaciones
  • Especifica comportamiento
  • Establece contratos
Implementación (Cómo lo hace)
  • Estructura de datos
  • Algoritmos específicos
  • Detalles internos

Ventaja clave: Una misma TDA puede ser implementada utilizando distintas estructuras de datos, manteniendo la misma funcionalidad.

2.3 Ejemplos de TDA Fundamentales

Muchos de los TADs más importantes son contenedores de datos, es decir, agregaciones de múltiples datos en una unidad conceptual:

Listas

Colecciones secuenciales de elementos

Pilas (Stack)

Colecciones que siguen la propiedad LIFO (Last In, First Out)

Colas (Queue)

Colecciones que siguen la propiedad FIFO (First In, First Out)

Árboles

Tipos arborescentes que representan colecciones de datos en una jerarquía

2.4 Implementación del TDA

Proceso de Implementación

1. Encontrar una Estructura de Datos (ED) adecuada

Para representar el modelo subyacente

2. Escribir procedimientos o funciones

Que cumplirán con las tareas definidas por las operaciones

Paradigma OOP: El paradigma de orientación a objetos facilita esto mediante el encapsulamiento de datos y operaciones a través de clases e interfaces, ocultando los detalles de implementación.

3. Representación en Memoria

3.1 Clasificación General

Las estructuras de datos se pueden clasificar, según su organización en memoria, en dos categorías principales:

Estática

Arreglos/Vectores

Memoria contigua

Dinámica

Nodos/Enlaces

Punteros

Esta elección tiene implicaciones directas en el coste temporal (Big-O) y espacial de las operaciones.

3.2 Representación Estática (Arreglo/Vector)

La representación estática implica que el espacio de almacenamiento se conoce en tiempo de compilación y se mantiene reservado durante la ejecución.

Características Principales

Contigüidad

Los elementos se almacenan en ubicaciones de memoria contiguas

Acceso Directo

Permite representación de acceso directo con coste O(1) para insertar, modificar, obtener y borrar

Limitación

Redimensionar requiere copiar todos los n elementos → O(n)

3.3 Representación Dinámica (Nodo/Enlace)

La representación dinámica permite que el espacio de almacenamiento se obtenga durante la ejecución, a medida que se necesita.

Características Principales

Nodos y Punteros

Se usa memoria dinámica con punteros que direccionan células en la memoria

Estructura Dinámica

La cantidad máxima de memoria ocupada no se conoce hasta que finaliza la ejecución

Coste en Tiempo

Operaciones como añadir, buscar y borrar generalmente tienen coste O(n) en el peor caso

3.4 Comparación: Arreglo vs. Enlace

Aspecto Arreglo/Vector Lista Enlazada Árbol AVL
Coste en Espacio O(MAX) - Fijo O(n) - Dinámico O(n)
Coste en Tiempo O(1) O(n) O(log n)
Flexibilidad Menos flexible Más flexible Excelente balance
Redimensionamiento Costoso O(n) No necesario Automático

3.5 Ventajas y Desventajas

Representación Estática

✅ Ventajas
  • Acceso directo O(1)
  • Menor overhead de memoria
  • Mejor localidad de referencia
❌ Desventajas
  • Tamaño fijo
  • Redimensionamiento costoso
  • Posible desperdicio de memoria

Representación Dinámica

✅ Ventajas
  • Tamaño flexible
  • Uso eficiente de memoria
  • No limitación de cardinal máximo
❌ Desventajas
  • Acceso secuencial O(n)
  • Overhead de punteros
  • Fragmentación de memoria

4. Síntesis y Conclusiones

4.1 Conceptos Clave Aprendidos

Notación Big-O

Herramienta fundamental para analizar y comparar la eficiencia de algoritmos y estructuras de datos

Tipos de Datos Abstractos

Separación clara entre interfaz e implementación, permitiendo flexibilidad y reutilización

Representación en Memoria

La elección entre estática y dinámica impacta directamente en rendimiento y flexibilidad

4.2 Aplicación Práctica

Criterios de Selección

Usar Arreglos cuando:
  • Acceso frecuente por índice
  • Tamaño conocido y fijo
  • Rendimiento crítico
Usar Listas Enlazadas cuando:
  • Tamaño variable
  • Inserciones/eliminaciones frecuentes
  • Memoria limitada
