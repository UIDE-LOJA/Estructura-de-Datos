Esta actividad práctica te permitirá aplicar los conceptos fundamentales de análisis de complejidad Big-O mediante el estudio de algoritmos JavaScript reales. Te enfocarás en contar operaciones, identificar patrones de crecimiento, y clasificar algoritmos según su eficiencia usando el método aprendido en clase.

¿Qué realizarás?

Actividad Instrucciones específicas Entregable
Preparación individual Revisa el ejemplo práctico de la presentación (Búsqueda del máximo). Descarga los 6 archivos JavaScript proporcionados por el docente. Comprensión del método de análisis y archivos descargados.

Algoritmos a analizar (proporcionados por el docente):

Archivo Algoritmo Descripción
algoritmo-01 Búsqueda Lineal Busca un elemento específico recorriendo la lista secuencialmente
algoritmo-02 Acceso Directo Obtiene un elemento en una posición específica del array
algoritmo-03 Suma de Elementos Calcula la suma total de todos los elementos de una lista
algoritmo-04 Comparar Pares Compara cada elemento con todos los demás elementos (bucles anidados)
algoritmo-05 Encontrar Mínimo Encuentra el elemento más pequeño en una lista de números
algoritmo-06 Operación Constante Realiza operaciones que no dependen del tamaño de la entrada

Método de análisis (seguir este proceso para cada algoritmo):

  1. Identificar operaciones principales: Bucles, comparaciones, asignaciones, accesos a array
  2. Contar ejecuciones: ¿Cuántas veces se ejecuta cada operación en función de n?
  3. Determinar el peor caso: ¿Cuál es el escenario que más operaciones requiere?
  4. Crear fórmula matemática: Expresar el total de operaciones como función de n
  5. Simplificar: Ignorar constantes y coeficientes, quedarse con el término dominante
  6. Clasificar Big-O: O(1), O(log n), O(n), O(n log n), O(n²), etc.
