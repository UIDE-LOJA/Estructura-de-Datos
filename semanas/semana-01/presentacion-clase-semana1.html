Notación Big-O

El Lenguaje Universal: Big-O

Para poder hablar de rendimiento, necesitamos un lenguaje común

Ese lenguaje universal es la notación Big-O

Importante: No mide tiempo en segundos o minutos. Mide algo mucho más crucial: cómo crece el problema

¿Qué es Big-O?

Es una forma de predecir cómo se va a comportar nuestro código en el peor de los casos

A medida que la cantidad de datos (n) se hace más y más grande,
¿cuánto trabajo extra tiene que hacer el algoritmo?

¿El doble? ¿1000 veces más? Big-O nos da esa respuesta

Como dice Héctor de León: "Es algo fundamental en el día a día de un profesional"

Ejemplo Práctico: Cómo Clasificar un Algoritmo

Algoritmo: Buscar el máximo en una lista

Código JavaScript 
function buscarMaximo(lista) {
    let maximo = lista[0];
    
    for (let i = 1; i < lista.length; i++) {
        if (lista[i] > maximo) {
            maximo = lista[i];
        }
    }
    
    return maximo;
}
Ejemplo de uso:
const numeros = [3, 7, 2, 9, 1, 5];
const resultado = buscarMaximo(numeros);
// resultado = 9
Paso 1: Contar Operaciones
Identificar cada operación:
  • let maximo = lista[0]: 1 operación
  • Bucle for: se ejecuta n-1 veces
  • Comparación if: n-1 comparaciones
  • Asignación maximo: ocasional
  • return maximo: 1 operación
Demo Interactivo

Abre la consola del navegador (F12) y haz clic

Clave: Identifica el bucle más profundo y cuenta cuántas veces se ejecuta en función de n

Total de Operaciones

Sumando todas las operaciones

Desglose de operaciones:
Inicialización: 1 operación
+
Bucle for: (n-1) iteraciones
+
Comparaciones if: (n-1) comparaciones
=
Fórmula Final

1 + (n-1) + (n-1) = 2n-1

Esta es nuestra fórmula matemática base

Ejemplo: Para una lista de 6 elementos: 2(6)-1 = 11 operaciones

Paso 2: Simplificar y Clasificar

De la fórmula matemática a Big-O

Proceso de Simplificación

Fórmula inicial: 2n - 1

↓

Ignorar constantes: 2n - 1 ≈ 2n

↓

Ignorar coeficientes: 2n ≈ n

Clasificación Final

Complejidad Temporal: O(n)

Complejidad Espacial: O(1)

Lineal en tiempo, constante en espacio

¿Por qué O(1) en espacio? Solo usamos una variable extra (maximo), sin importar el tamaño de la lista

Tipos de Datos Abstractos (TDA)

Los Planos para Construir la Solución

Si Big-O es la herramienta para medir el problema (el termómetro)

Los TDA son los planos para construir la solución

Son como los bocetos, las ideas que nos dicen
cómo organizar la información para que sea eficiente

Como el manual de usuario de un coche: Te dice lo que puedes hacer (acelerar, frenar, girar), pero no te explica la ingeniería del motor

Características Fundamentales

Abstracción

Se centra en la abstracción de datos, ocultando detalles internos

Interfaz

Encapsulamiento

Oculta los detalles de implementación del usuario final

Ventaja: Separación entre interfaz (qué) e implementación (cómo)

Pila (Stack) - LIFO Interactivo

LIFO - Last In, First Out

El último que entra es el primero que sale

Como una pila de platos: solo puedes coger el de arriba

Visualización de la Pila
La pila está vacía
↑ TOP (último elemento agregado)

Operaciones: Push (agregar arriba) | Pop (quitar de arriba) | Top (ver el de arriba)

Cola (Queue) - FIFO Interactivo

FIFO - First In, First Out

El primero que llega es el primero al que atienden

Como la cola del banco: el primero en llegar, primero en ser atendido

Visualización de la Cola
FRENTE (primero en salir) ← ← ← FINAL (último en entrar)
La cola está vacía

Operaciones: Encolar (agregar al final) | Desencolar (quitar del frente) | Frente (ver el primero)

Representación en Memoria

Ahora Toca Ponerse el Casco y Construir

Ya tenemos la forma de medir (Big-O) y los planos (TDA)

La obra se hace en un sitio muy concreto: la memoria del ordenador

La Primera Gran Decisión de la Obra

¿Construimos con ladrillos fijos (estática) o usamos algo más flexible (dinámico)?

Memoria Estática: Los Arrays

Es como reservar una mesa entera en un restaurante,
aunque no sepas cuántos van a venir

Se reserva un bloque fijo de espacio desde el principio

El Superpoder de los Arrays

Como todos sus elementos están juntitos en memoria, uno al lado del otro,
son como vecinos en una calle

El ordenador puede hacer un cálculo súper simple y ¡zas!
Sabe exactamente dónde está el elemento número 500

Resultado: Acceso casi instantáneo O(1)

La Criptonita del Array

Todo superpoder tiene su criptonita

Esa rigidez que lo hace tan rápido es también su talón de Aquiles

¿Qué pasa si hemos calculado mal el tamaño de la mesa?

¿Qué ocurre cuando nuestro array se llena hasta los topes?

Un Auténtico Drama para el Rendimiento

1. El sistema tiene que buscar otro sitio en la memoria, un bloque nuevo y más grande

2. Luego tiene que copiar todos y cada uno de los elementos, uno por uno

De repente, añadir un elemento más se convierte en un coste O(n)

La Alternativa: Construcción Dinámica

Es más como ir pidiendo sillas a medida que llega la gente

Va pidiendo pequeños trozos de memoria solo cuando los necesita,
sobre la marcha

La Idea es Diferente

Cada nodo es como una cajita que tiene dos cosas:

El dato en sí

Una flecha (puntero)
que dice dónde está la siguiente cajita

Es como una gincana

Flexibilidad vs Coste

La Gran Ventaja

Flexibilidad total

Añadir o quitar una cajita nueva es súper fácil y no molesta para nada el resto de la cadena

El Coste

En la vida no hay nada gratis

No podemos saltar directamente a un elemento. Hay que empezar por el principio y seguir las flechas una a una

En el peor caso: O(n)

El Momento de la Verdad

Con todo esto sobre la mesa, llegamos al punto donde todas las piezas del puzzle encajan

La medición del rendimiento + los planos de los TDA + la construcción real en memoria

Esta tabla lo resume todo a la perfección

Necesidad Solución Ganadora Rendimiento
Acceso a la velocidad del rayo Array (si sabes el tamaño) O(1)
Flexibilidad total Lista Enlazada O(n)
Lo mejor de los dos mundos Árboles Balanceados O(log n)

La Gran Conclusión

La decisión que tomas a bajo nivel determina directamente el rendimiento Big-O

La decisión de cómo representar los datos en memoria:
¿Con un bloque compacto (array) o con una cadena de nodos?

Aquí es donde la teoría y la práctica se dan la mano. Son inseparables.

La Pregunta de Verdad

La próxima vez que un código empiece a ir lento, a ahogarse...

La pregunta ya no será: "¿Cómo arreglo esto?"

La pregunta de verdad, la importante será:

"¿Qué decisión tomé sobre la estructura de datos que me ha traído hasta este punto?"

Ahora que conoces las herramientas, la pregunta es: ¿Cómo vas a construir tu próximo código?

Ejemplo Resuelto: Algoritmo 1

Búsqueda Lineal - Análisis Completo

Código JavaScript 
function busquedaLineal(lista, elemento) {
    for (let i = 0; i < lista.length; i++) {
        if (lista[i] === elemento) {
            return i;
        }
    }
    return -1;
}
Ejemplo: 
const numeros = [5, 2, 8, 1, 9, 3];
const resultado = busquedaLineal(numeros, 8);
// resultado = 2
Análisis Big-O
Conteo de operaciones:
  • Bucle for: se ejecuta hasta n veces
  • Comparación if: 1 por iteración
  • Return (si encuentra): 1 operación
  • Return (si no encuentra): 1 operación
Peor caso:

Elemento no está en la lista

Fórmula: n comparaciones + 1 return

Total: n + 1 operaciones

Clasificación Final

Tiempo: O(n)

Espacio: O(1)

Justificación: En el peor caso debe revisar todos los elementos (n), por lo tanto es O(n). El espacio es O(1) porque solo usa variables auxiliares constantes.
